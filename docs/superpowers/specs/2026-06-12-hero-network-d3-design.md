# 首頁 Hero 淡網絡 d3 特效 — 設計文件

- 日期：2026-06-12
- 範圍：首頁 `index.astro` 的 `.intro`（`#main > section > div` Hero 區）
- 目標：在保留現有「卡片＋長條圖＋金勾」前景的前提下，於背景加入一張用 d3 驅動的節點網絡，作為延續品牌語彙（亞太專業觀點＝串連可信網絡）的氛圍主視覺。

## 核心原則

靜止時就是一張**清晰、持續流動**的深藍網絡，並有**少數金色節點恆亮**（像幾個被點亮的專業觀點）。滑鼠互動是「再加一層」金色回饋，而非網絡唯一的存在感。網絡是背景，不搶前景卡片與兩顆 CTA。

## 視覺與行為

### 構圖
- `.intro` 區滿版鋪一張節點網絡 `<canvas>`，墊在所有內容後方。
- 現有卡片與左側文字維持原樣，浮在前景。

### 密度漸層（左淡右密）
- 約 55–65 個節點（桌機）。
- 節點出現機率由左到右遞增：左側文字後方近乎留白，右側卡片後方最密，形成自然視覺重心。

### 靜止樣態（明顯可見）
- 節點：1.5–2.5px 小圓點，深藍（`--appi-brand`）。
- 基準透明度約 **0.35–0.6**（由左到右遞增）——靜止時網絡明顯可見、看得出是一張持續流動的網。
- 連線：鄰近節點（距離 < ~110px）之間浮現細線，透明度隨距離衰減；節點漂移時連線自然淡入淡出，產生「星座呼吸」感。

### 漂移
- 每個節點環繞自己的「家」位置極慢游移，永不靜止，但速度刻意壓低、不躁動。

### 金色（恆亮 ＋ 互動，兩者並存）
- **恆亮**：靜態時固定 2–3 個金色節點（`--appi-accent`），帶極輕微脈動讓它「活著」；其鄰近連線也微微帶金。
- **互動回饋**：游標靠近（半徑 ~120px）時，附近節點被輕輕吸引靠攏、連線變亮並短暫泛金；游標移開後在 ~0.6s 內柔順褪回淡深藍。
- 金色與卡片上那顆金勾遙相呼應，但不重複堆疊。

## 技術架構

### 新元件
- `src/components/blocks/HeroNetwork.astro`。
- 在 `index.astro` 的 `.intro` 內、`.intro-inner` 之前插入。

### 渲染
- `<canvas>` + 2D context + `requestAnimationFrame`。
- 節點數少，但 canvas 比 SVG 更省、漸層透明度更好控。

### d3 用途（真正用到 d3）
- `d3-force`：驅動節點運動
  - `forceX` / `forceY`：把每個節點輕拉回各自「家」座標（維持密度漸層）。
  - 微弱 `forceManyBody`：維持節點間距、避免結團。
  - 自訂 pointer force：滑鼠靠近時對附近節點施加輕微吸引。
  - `alphaTarget` 保持極小值（搭配低 `alphaDecay`），讓模擬永遠溫和流動、不凍結。
- `d3-quadtree`（隨 `d3-force` 附帶）：每幀高效找鄰近節點以畫連線。
- `d3-scale` + `d3-interpolate` / `d3-color`：密度漸層、距離→透明度、深藍↔金色色彩內插。

### 依賴
- 模組化安裝：`d3-force`、`d3-scale`、`d3-interpolate`、`d3-color`（不裝整包 `d3`，控制 bundle 大小）。

### 層次與點擊
- `.intro` 設 `position: relative`。
- canvas：`position: absolute; inset: 0; pointer-events: none`（不搶 CTA 點擊）。
- 滑鼠座標改在 `.intro` 上監聽 `mousemove`，換算成 canvas 內座標餵給 pointer force。
- `.intro-inner` 設 `position: relative; z-index: 1`。
- 卡片白底不透明，乾淨壓在網絡之上。

### 色彩來源
- 從 CSS 變數讀取實際品牌色（`--appi-brand`、`--appi-accent`），確保亮／暗色模式一致。
- 本站 CSS 變數實際解析為 `oklch()`，而 d3 `interpolateRgb` 不解析 oklch。故初始化時以 1×1 離屏 canvas 將色值**點陣化成具體 rgb** 再餵給 d3（跨色彩格式最穩，無效時回退 hex）。

## 效能 / 無障礙 / RWD

### `prefers-reduced-motion: reduce`
- 不啟動動畫迴圈，改畫一張靜態網絡單幀（含恆亮金色節點），尊重使用者偏好。

### 離開視窗暫停
- `IntersectionObserver` 監看 `.intro`；捲出畫面時停掉 `requestAnimationFrame`，捲回再恢復，省電。

### DPR / resize
- 依 `devicePixelRatio` 縮放 canvas 保持銳利。
- 視窗縮放時重算節點「家」座標與密度分布。

### 行動版
- ≤900px：版面堆疊、插畫縮小；節點數下修（約 30），關閉滑鼠互動（觸控無 hover），維持明顯但更輕的網絡背景。
- ≤520px：必要時直接靜態單幀或隱藏網絡，避免效能與雜訊。

## 不做（YAGNI）
- 不做滑鼠視差、不做點擊新增節點、不做資料驅動（節點不對應真實文章資料），純氛圍視覺。
- 不替換現有卡片插畫；卡片維持原樣。

## 驗收標準
1. 桌機：Hero 靜止時可清楚看見一張持續流動的深藍網絡，左淡右密，並有 2–3 個恆亮金色節點。
2. 滑鼠滑過右側時，附近節點靠攏、連線與節點短暫泛金，移開順暢褪回。
3. 兩顆 CTA（探索所有文章 / 成為作者）點擊不受 canvas 影響。
4. `prefers-reduced-motion` 下無動畫、僅靜態單幀。
5. Hero 捲出畫面後 rAF 停止運作。
6. 行動版網絡變輕、互動關閉，不影響可讀性與效能。
